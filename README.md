# Mobile Crisis Services in Eugene OR: Call Outcome Analysis

## Overview

This repository contains data and scripts for the CAHOOTS and MCS LC data analysis project, focused on analyzing outcomes of calls and intervention using emergency response data.

## Project Structure

```
CAHOOTS/
├── biblio/                # Bibliographic references and related documents
├── data/                  # All data files
│   ├── raw/               # Raw, unprocessed data
│   ├── clean/             # Cleaned data
│   ├── intermediate/      # Intermediate data files (e.g., RDS)
│   ├── processed/         # Final processed datasets
│   └── samples/           # Sample datasets for quick exploration
├── figures/               # Figures and plots generated by analyses
├── renv/                  # R environment and package management
├── report/                # Reports and manuscripts
├── results/               # Output results from analyses
├── scripts/               # All R scripts, organized by workflow step
│   ├── 1_convert_to_csv/  # Scripts for data conversion
│   ├── 2_cleaning/        # Data cleaning scripts
│   ├── 3_mapping/         # Data mapping and exploration
│   ├── 4_analysis/        # Analysis scripts
│   └── 5_modeling/        # Modeling scripts
├── CAHOOTS.Rproj          # RStudio project file
├── LICENSE                # License information
├── PROJECT_DESCRIPTION.md # Project description and metadata
└── README.md              # This file
```

## Getting Started

### Prerequisites
- [R](https://cran.r-project.org/) (version 4.5 or higher recommended)
- [RStudio](https://posit.co/download/rstudio-desktop/) (optional, but recommended)

### Clone the Repository
```sh
git clone <https://github.com/mathilde-ly/CAHOOTS.git>
cd CAHOOTS
```

### Managing Dependencies
This project uses [`renv`](https://rstudio.github.io/renv/) for R package management. To install all required packages:

1. Open the project in RStudio or start an R session in the project directory.
2. Run:
	```R
	renv::restore()
	```
	This will install all packages listed in the project's lockfile.

If you add new packages, use `renv::snapshot()` to update the lockfile.

## Running the Project

Scripts are organized by workflow step in the `scripts/` directory. Typical workflow:

1. **Convert raw data to CSV:**
	- Run scripts in `scripts/1_convert_to_csv/` to merge and convert raw data files.
2. **Clean data:**
	- Use scripts in `scripts/2_cleaning/` to clean and preprocess data.
3. **Map and explore data:**
	- Explore and map call outcomes in data using scripts in `scripts/3_mapping/`.
4. **Analysis and modeling:**
	- Perform analyses in `scripts/4_analysis/` and modeling in `scripts/5_modeling/`.

Each script is documented with usage instructions. Run scripts in order for a full pipeline, or individually as needed.

## Adding or Updating Data
- Place new raw data files in `data/raw/`.
- Update or add scripts as needed in the appropriate `scripts/` subdirectory.

## Reproducibility
All package versions are pinned using `renv`. Always run `renv::restore()` after pulling changes.
